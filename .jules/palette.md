## 2024-05-14 - Header Layout Interactive Elements Missing ARIA Labels
**Learning:** Found an accessibility pattern in the main layout (`Header.tsx`) where icon-only buttons (Menu toggle, Notifications) lacked `aria-label` attributes and screen reader context for icons (missing `aria-hidden="true"`). The search input also lacked an accessible name. Without these labels, screen reader users would only hear "button" without understanding its purpose. In addition, interactive elements were missing visible focus states for keyboard users.
**Action:** Always verify that every interactive element that doesn't have visible text content (like an icon) receives a descriptive `aria-label`. Decorative icons should always have `aria-hidden="true"`. Also ensure that custom styled inputs have `aria-label`s when visual labels are omitted, and always implement `focus-visible` styles on interactive elements to support keyboard navigation.
