## 2024-07-13 - [Sentinel] Reverse Tabnabbing Vulnerability
**Vulnerability:** External links created with `target="_blank"` both in standard JSX (`src/pages/admin/Settings.tsx`) and dynamically generated via `document.createElement('a')` (`src/pages/admin/DocumentRepository.tsx`) lacked `rel="noopener noreferrer"`.
**Learning:** React does not automatically add `rel="noopener noreferrer"` to dynamic links created via native DOM APIs (e.g. `document.createElement`), nor does it enforce it on all static tags without proper ESLint rules configuration. This exposes users to reverse tabnabbing attacks, where the newly opened tab can control the `window.opener` object to redirect the original tab to a malicious site.
**Prevention:** Always add `rel="noopener noreferrer"` to anchor tags when using `target="_blank"`, especially when manipulating the DOM directly or downloading files from external URLs. Configure ESLint rules (`react/jsx-no-target-blank`) to enforce this on JSX elements.
