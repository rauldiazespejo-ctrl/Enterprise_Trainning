## 2026-06-17 - Sistema de Audit Log ISO 45001

### Lección Reusable: Patrón de Audit Log en React Context + Supabase Edge Functions

**Contexto:** Implementación de audit log para cumplimiento ISO 45001 en la plataforma.

**Patrón aplicado:**

```typescript
// Helper reutilizable para audit log
const logAuditEvent = async (
  action: string,
  resourceType: string,
  resourceId?: string,
  details?: Record<string, unknown>
): Promise<void> => {
  try {
    await supabase.functions.invoke('audit-log', {
      body: { action, resource_type: resourceType, resource_id: resourceId, details }
    });
  } catch (err) {
    console.warn('[Audit] Failed to log event:', err);
  }
};
```

**Reglas clave:**
1. Usar `void` al llamar (fire-and-forget) para no bloquear la UI
2. El helper debe capturar errores silenciosamente — nunca romper el flujo del usuario por un fallo de logging
3. Pasar `user_id` cuando esté disponible, `null` si es anónimo
4. Incluir detalles contextuales en JSONB para auditoría ISO 45001

**Archivos del patrón:**
- Migración: `supabase/migrations/YYYYMMDDHHMMSS_create_audit_log.sql`
- Edge Function: `supabase/functions/audit-log/index.ts`
- Integración: contextos React que llaman `logAuditEvent()`
